スペシャルユーザーGUI概念メモ

旧 ncv_special_monitor のGUIから、新しい監視アプリへ引き継ぐ画面階層と操作概念を固定する。 これは実装前の設計メモであり、まず「どの画面から何をダブルクリックして、どの編集画面へ進むか」を明確にする。

画面遷移

1. スペシャルユーザーメイン画面 登録済みスペシャルユーザーを一覧表示する。 ユーザー行をダブルクリックすると、スペシャルユーザー編集画面を開く。
2. スペシャルユーザー編集画面 対象ユーザーの基本情報、AI分析、既定応答、スペシャルトリガー、配信者一覧を編集する。 配信者一覧の行をダブルクリックすると、配信者編集画面を開く。
3. 配信者編集画面 そのスペシャルユーザーに紐づく特定配信者向けの設定を編集する。 配信者ごとのトリガー一覧を持ち、トリガー行をダブルクリックするとトリガー編集画面を開く。
4. トリガー編集画面 トリガー名、キーワード、応答タイプ、定型メッセージ、AIプロンプト、反応回数、遅延などを編集する。

階層構造

階層 主な対象 主な操作 次に開く画面
スペシャルユーザー一覧 スペシャルユーザーID、表示名、AIモデル、配信者数、スペシャルトリガー数 追加、編集、削除、配信者管理、シリーズ管理、行ダブルクリック スペシャルユーザー編集
スペシャルユーザー編集 ユーザー基本情報、184別名ID、AI分析、既定メッセージ応答、スペシャルトリガー、配信者一覧 保存、キャンセル、配信者追加/削除、一括有効/無効、配信者行ダブルクリック 配信者編集
配信者編集 配信者ID、配信者名、有効状態、この配信者向けのAI分析/応答設定、トリガー一覧 保存、キャンセル、トリガー追加/削除/シリーズ追加、一括有効/無効、トリガー行ダブルクリック トリガー編集
トリガー編集 トリガー名、有効状態、キーワード、応答タイプ、定型メッセージ、AIプロンプト、反応回数、遅延 保存、キャンセル なし

重要な概念

  • スペシャルユーザーは監視対象のコメント投稿者である。
  • スペシャルユーザーには、出入りする配信者を複数紐づけられる。
  • 配信者ごとに、その配信内でだけ有効な応答・AI・トリガー設定を持てる。
  • トリガーはキーワード条件に一致した時だけ発動する個別ルールである。
  • スペシャルトリガーは通常設定より優先されるユーザー直下の特別ルールである。
  • 各階層の一覧は、行ダブルクリックで下位または詳細編集画面へ入る。

新アプリ側での対応方針

旧GUI概念 新PyQtアプリでの予定
スペシャルユーザー一覧 現在の「スペシャルユーザー」タブを拡張する。
スペシャルユーザー編集 ダブルクリックでサブウィンドウを開き、基本情報、既定アクション、トリガーワード、配信者一覧を編集する。
配信者一覧/配信者編集 スペシャルユーザーに紐づいた放送者IDを一覧化し、配信者ごとの設定画面を追加する。

投稿/外部アプリ実行 まず設定保存だけ実装し、実行処理は後で安全に接続する。

ニコニコ投稿セッション取得メモ

通常コメント投稿は、Chrome拡張「ニコ生コメビュ」v0.3.4 の実装を参考にして、Python側で直接送信できることを確認済み。 拡張機能を実行しているのではなく、拡張の通信手順を読み取って nicolive_post.py に移植している。

項目 内容
必要なCookie user_session
取得元 ニコニコにログイン済みのChromeプロファイル。別アカウントで投稿したい場合は、そのアカウントでログインしているChromeプロファイルから取得する。
保存先DB data/tracker.db の niconico_sessions テーブル。
現在の使用ルール nicolive_post.py は updated_at DESC, id DESC で最新の user_session を使う。
184投稿 isAnonymous: true を送る。成功時はサーバー応答の postCommentResult.data.chat.mail が 184、anonymity が 1 になる。

セッションIDの取得手順

  1. 投稿に使いたいアカウントでChromeにログインし、ニコニコにログインしておく。
  2. 新アプリの「設定」タブを開く。
  3. ログイン用Chromeを開く を押し、必要ならそのChromeでニコニコログイン状態を確認する。
  4. ログイン完了後に取得 を押す。
  5. user_session 欄に user_session_... が入ったことを確認する。
  6. 識別用のアカウント名を入れて保存する。例: 応援アカウント、youzoom。

手動で確認する場合

  1. Chromeで https://www.nicovideo.jp/ または対象放送ページを開く。
  2. DevToolsを開き、Application タブを開く。
  3. Storage → Cookies → https://www.nicovideo.jp または https://live.nicovideo.jp を開く。
  4. Cookie一覧から user_session を探す。
  5. 値はアカウントのログイン権限そのものなので、ログや画面に不用意に出さない。

実証済み投稿フロー

  1. DBから user_session を読む。
  2. https://live.nicovideo.jp/watch/lv... をCookie付きで取得する。
  3. ページ内の embedded-data から認証済み webSocketUrl を読む。
  4. WebSocketへ接続し、startWatching を送る。
  5. messageServer イベントから vposBaseTime を受け取る。
  6. 現在時刻との差分から vpos を計算する。
  7. postComment を送る。
  8. postCommentResult が返れば投稿受付成功。
