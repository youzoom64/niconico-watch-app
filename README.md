# niconico-watch-app 取扱説明書

ニコニコ生放送を監視し、録画、コメント保存、文字起こし、要約、画像生成、HTMLアーカイブ作成、公開までを行うWindows用GUIアプリです。

この説明書はGUIでの利用を前提にしています。コマンド操作は通常不要です。

## このアプリでできること

- 登録した配信者の放送開始を監視する
- 放送を録画し、録画が途切れた場合は再開する
- コメントを取得して放送ごとに保存する
- Faster-WhisperまたはWhisperXで文字起こしする
- WhisperXで話者を分ける
- 要約、感情分析、単語分析、スクリーンショット、要約画像を作る
- 放送ごとのHTMLと配信者別の一覧ページを作る
- 人物タグから、その人物が話題になった時刻へ移動する
- 作成したHTMLを設定済みのWebサーバーへ送る
- 保存済みのMP4/MKVを後から処理し直す

## 1. 事前に用意するもの

### 必須

- Windows 10または11
- インターネット接続（初回セットアップ時）

Python、FFmpeg、FFprobeを事前にインストールする必要はありません。`setup.bat`がPython 3.11をプロジェクト内の`tools\python311`へ取得し、専用の`.venv`を作成します。FFmpeg一式もプロジェクト内へ自動取得します。

既に別のPythonや共通venvが入っていても使用しません。配布先のPCにPythonが一切入っていない状態からセットアップできます。

### 録画する場合

`SlNicoLiveRec`の手動ダウンロードや実行ファイルの指定は不要です。`start.bat`が公式配布元の最新版を確認し、未導入なら自動でダウンロード・展開して、niconico-watch-appへ実行ファイルの場所を設定します。

更新版が公開されている場合も、起動時に最新版を取得します。以前のSlNicoLiveRec設定がある場合は、ログイン設定を新しい版へ引き継ぎます。

- 公式配布元：[SlNicoLiveRec配布ページ](https://person-of-ehomaki.blog.jp/)
- SlNicoLiveRecの著作権その他の権利は、SlNicoLiveRecの開発・配布者に帰属します。
- このアプリはSlNicoLiveRec本体を同梱・改変再配布せず、公式配布元から公開されているファイルを自動取得します。

既に保存済みのMP4/MKVからHTMLを作るだけなら、SlNicoLiveRecは不要です。

#### SlNicoLiveRecへ自動適用される設定

導入時に次の推奨設定が自動適用されます。通常はSlNicoLiveRecを手動で設定する必要はありません。

| 項目 | 設定値 |
|---|---|
| ファイル名の書式を変更する | ON |
| Filename | `{id}_{year}_{month}{day}_{hour}{minute}{second}_{title}` |
| フォルダ名の書式を変更する | ON |
| FolderName | `{supplier_id}_{author}` |
| タイトルバー書式 | `{author} - SlNicoLiveRec` |
| 録画停止後に再録画を試みる | OFF |
| 再録画試行待機（秒） | `10` |
| 試行上限 | `0` |
| 開演待機・指定秒数まで待機する | OFF |
| 秒前まで待機 | `60` |
| 自動終了時にウィンドウを閉じる | ON |
| Debug mode | OFF |

「設定 → FFmpeg」は次のように設定します。

| 項目 | 設定値 |
|---|---|
| 録画終了後にフォーマットをMP4に変換 | ON |
| 変換後にオリジナルのTSファイルを削除 | ON |
| TSファイル削除時にゴミ箱に移動 | ON |
| 変換オプション | `-c:v copy -c:a copy` |

録画用の`user_session`は、niconico-watch-appの設定画面にある「録画用 user_session」へ入力して保存します。保存時にSlNicoLiveRecの設定へ自動反映されます。

`user_session`はログイン資格情報です。スクリーンショット、README、GitHub、チャットへ貼り付けないでください。

#### user_sessionの取得方法

通常はniconico-watch-appのGUIから取得できます。

1. メインGUIの「基本設定」を開きます。
2. ニコニコアカウント欄の「ログイン用Chromeを開く」を押します。
3. 開いたChromeで、録画に使うニコニコアカウントへログインします。
4. GUIへ戻り「ログイン完了後に取得」を押します。
5. `user_session`欄へ値が入ったことを確認します。
6. 「保存」を押します。
7. 録画に使う値は「録画用 user_session」へ入力します。SlNicoLiveRec側へ手動入力する必要はありません。

GUIで取得できない場合は、ログイン済みChromeでニコニコを開き、`F12`で開発者ツールを表示します。`Application` → `Storage` → `Cookies` → `https://www.nicovideo.jp`を開き、Cookie一覧の`user_session`にあるValueを使用します。Chromeの表示言語によっては`Application`が「アプリケーション」と表示されます。

ログアウトやセッション更新後は古い値が無効になることがあります。録画やコメント取得でログインエラーが出た場合は、同じ手順で取り直してください。

### GPUで文字起こしする場合

- NVIDIA製GPU
- 対応するNVIDIAドライバー
- CUDA対応版PyTorch

### 話者分離する場合

- Hugging Faceアカウント
- Hugging Faceトークン
- 使用するpyannoteモデルの利用条件への同意

## 2. インストール

1. GitHubページ上部の「Code」を押します。
2. 「Download ZIP」を押します。
3. ダウンロードしたZIPを任意のフォルダへ展開します。
4. 展開したフォルダー直下の`setup.bat`をダブルクリックします。
5. Python仮想環境、必要ライブラリ、FFmpegの準備が完了するまで待ちます。
6. 以後はフォルダー直下の`start.bat`をダブルクリックして起動します。

`start.bat`は、プロジェクト内の`.venv`または同梱用FFmpegが見つからない場合、`setup.bat`を自動実行します。共通venvや別フォルダーのPython環境は使用しません。

初回セットアップでは必要なライブラリをダウンロードするため、時間がかかります。

「Python 3.11 is not installed」と表示された場合は、Python 3.11をインストールしてから再実行してください。

## 3. アプリを起動する

通常は、フォルダー直下の`start.bat`をダブルクリックして起動してください。必要な環境がまだ準備されていない場合はセットアップも自動的に行うため、初めて使う場合も以後の日常的な起動も`start.bat`だけで構いません。

`scripts\start_gui.cmd`は、セットアップ済みのGUIを直接起動したい開発・保守作業向けです。通常利用では使う必要はありません。

黒い画面が開いた後、メインGUIが表示されます。黒い画面はログ表示とプロセス管理に使うため、GUIの利用中は閉じないでください。

タイムシフト処理だけを使う場合は`scripts\start_timeshift_gui.cmd`をダブルクリックします。

## 4. 最初に行う設定

メインGUIの設定画面で、次を順番に設定します。

1. 保存先フォルダ
2. 録画用user_session（SlNicoLiveRec本体と実行ファイルの場所は自動設定）
3. 監視する配信者
4. 文字起こしエンジン
5. 実行する生成項目
6. 必要であれば公開先

設定や処理状況は`data\tracker.db`へ保存されます。通常はこのDBを直接編集しないでください。

放送データは次の場所へ保存されます。

```text
設定した保存先\platform\niconico\配信者ID\broadcast\放送ID\
```

旧版の`bloadcast`は綴り間違いです。現在は正しい`broadcast`へ統一されています。

## 5. 配信者を監視する

各設定項目の詳しい説明は、[配信者設定 取扱説明書をブラウザで見る](https://warehouse.bitter.jp/niconico-watch-app/docs/broadcaster-settings.html)を参照してください。

スペシャルユーザーの登録・AI分析・自動反応・配信者との紐付けは、[スペシャルユーザー設定 取扱説明書をブラウザで見る](https://warehouse.bitter.jp/niconico-watch-app/docs/special-user-settings.html)を参照してください。

1. メインGUIで監視配信者の編集画面を開きます。
2. ニコニコの配信者IDと表示名を入力します。
3. 録画、文字起こし、要約、画像など必要な機能へチェックを入れます。
4. 保存します。
5. 監視を開始します。

配信が始まると、録画とコメント取得を開始します。放送終了後、チェックした生成項目を順番に処理します。

## 6. 文字起こしを設定する

### Faster-Whisper

通常の文字起こし向けです。WhisperXより構成が単純で、話者分離が不要な場合に向いています。

### WhisperX

次が必要な場合に使用します。

- 単語単位の時刻
- 発言者の分離
- 詳細な話者ラベル

GPUを使う場合の基本設定は次です。

- デバイス：`cuda`
- 計算タイプ：`float16`
- モデル：精度優先なら`large-v3`

「CUDA out of memory」が出る場合は、最初にバッチサイズを下げてください。それでも不足する場合はビームサイズまたはモデルサイズを下げます。

### 初期プロンプト

初期プロンプトは「どのような音声なのか」を文字起こしエンジンへ伝える欄です。

空欄の場合は次の文章が自動で使われます。

> これはニコニコ生放送の録画音声です

固有名詞が多い放送では「Hotwordsに含まれる語は主に人物名・固有名詞です」と加えると、文脈を補助できます。

### 認識支援文字と正式タグ

監視配信者編集画面の「使用タグ候補」で設定します。

| 認識支援文字 | 正式タグ |
|---|---|
| 文字起こしで認識させたい表記 | HTMLに表示する正式表記 |

- 左側はHotwordsとして文字起こしへ渡されます。
- 右側は一覧ページや人物タグで使われます。
- 同じ名称でよければ左右を同じにします。
- 表記を直したい場合だけ左右を変えます。

## 7. 生成項目を選ぶ

GUIではStep01～15として表示されます。チェックを外したStepは実行されません。

| Step | 内容 |
|---|---|
| 01 | 放送情報を集める |
| 02 | 音声抽出と文字起こし |
| 03 | 感情分析 |
| 04 | 単語分析 |
| 05 | 要約 |
| 06 | 音楽生成 |
| 07 | 要約画像生成 |
| 08 | AI会話生成 |
| 09 | スクリーンショット生成 |
| 10 | コメント処理 |
| 11 | 特別ユーザーHTML生成 |
| 12 | 放送ごとのHTML生成 |
| 13 | タグと一覧データ生成 |
| 14 | 配信者別一覧ページ生成 |
| 15 | Webサーバーへアップロード |

## 8. 保存済み放送のHTMLを作り直す

文字起こしや要約をやり直さず、HTMLだけを作り直す手順です。

1. メインGUIの「確認」タブを開きます。
2. 対象の配信者を選びます。
3. 作り直したい放送の行を選びます。
4. 行を右クリックします。
5. 「ローカル処理へ送る」を押します。
6. タイムシフトGUIが開いたら、Step01～11のチェックを外します。
7. Step12・13・14だけをチェックします。
8. Webサーバーも更新する場合はStep15もチェックします。
9. 「ローカル動画からHTML作成」を押します。

複数の放送をまとめて選択して送ることもできます。

Step02を外している限り、文字起こしはやり直しません。DBに保存済みの文字起こし、コメント、要約、画像を再利用します。

「ローカル動画が見つかりません」と表示された場合は、対象のMP4またはMKVをタイムシフトGUIへ直接追加してください。ファイル名には`lv`から始まる放送IDが必要です。

## 9. ローカル動画・タイムシフトを処理する

1. `start_timeshift_gui.cmd`を起動します。
2. MP4またはMKVを画面へドラッグ＆ドロップします。
3. 必要なStepだけをチェックします。
4. 「ローカル動画からHTML作成」を押します。
5. 下部ログに完了と表示されるまで待ちます。

同じ放送IDの録画が複数に分かれている場合は、一つの放送として扱います。録画が途中で止まり、すぐ再開された場合も、DBの録画区間を使って時間軸を統合します。

## 10. 作成したHTMLを見る

メインGUIの確認タブから、対象放送の生成HTMLを開けます。

配信者フォルダには次が作られます。

- `index.html`：全放送の一覧
- `tags`フォルダ：タグ別一覧
- 放送IDフォルダ：個別HTML、文字起こし、音声、画像

個別HTML上部の人物タグを押すと、その人物が話題になった場所へ順番に移動します。発言中の人物名は正式タグ名へ勝手に置換せず、元の表記を残したまま回数だけを表示します。

スマホ専用HTMLは作成しません。通常のHTMLが画面幅に合わせて表示されます。

## 11. Webサーバーへ公開する

Step15を有効にすると、設定済み公開先へHTMLを送ります。

公開設定にはサーバー名、保存先、認証情報が必要です。認証情報をGitHubや生成HTMLへ書かないでください。

公開後は次を確認します。

- 一覧ページが開く
- 個別HTMLへのリンクが開く
- 画像が表示される
- タグ別一覧が崩れていない
- 個人情報やトークンが含まれていない

## 12. よくあるエラー

### 「venv Python not found」

`setup_venv.cmd`を先に実行してください。

### 「Torch not compiled with CUDA enabled」

CPU版PyTorchが入っています。使用するCUDAに合うPyTorchを入れ直す必要があります。

### 「CUDA failed with error out of memory」

文字起こしのバッチサイズを下げます。次にビームサイズを下げ、それでも不足する場合は小さいモデルを選びます。

### Hugging Faceの認証エラー

Hugging Faceトークン、モデルの利用条件への同意、ネット接続を確認してください。

### 話者ラベルが付かない

WhisperXだけでは話者分離されません。「話者分離」にチェックを入れ、Hugging Faceトークンを設定してください。

### Step12で動画が見つからない

保存先が次の形式になっているか確認してください。

```text
配信者ID\broadcast\放送ID
```

### GUIを閉じても処理が残る

タイムシフトGUIが開始した処理は、GUI終了時に停止する設計です。残る場合はタスクマネージャーでPython、FFmpeg、WhisperXを確認してください。監視アプリ本体は別プロセスです。

## 13. データとバックアップ

重要なデータは次です。

- `data\tracker.db`：放送、コメント、文字起こし、処理状態
- 設定した保存先：動画、音声、HTML、画像
- `config.json`：共通設定

アプリ更新前には、この3か所をバックアップしてください。

APIキー、Cookie、Hugging Faceトークン、FTPパスワードを公開リポジトリへ入れないでください。

## 14. 開発者向け

通常利用者はこの章を読む必要はありません。

アプリの入口は`main.py`です。GUI、タイムシフトGUI、トラッカー、ローカルAPIをサブコマンドで起動できます。テストは`pytest`を使用します。

メインGUIの「テスト」タブでは、`tests`ディレクトリにある高速テストを項目別または一括で実行できます。録画や外部APIを動かさず、LV番号、録画ファイル選択、保存先、設定の既定値、タグ変換、時間軸、公開URLなどの純粋処理を確認します。

開発時にコマンドから全件実行する場合は、プロジェクト直下で次を実行します。

```powershell
.\.venv\Scripts\python.exe -m pytest -q tests
```

主な補助ツール：

- `tools\run_broadcaster_step12_to_end.py`：配信者の保存済み全放送をStep12から一括処理
- `tools\regenerate_mobile_html.py`：旧スマホHTML互換ツール。通常運用では使用しません
- `tools\whisperx_transcribe_cli.py`：WhisperX文字起こし処理

## 注意事項

- ニコニコ生放送の利用規約、著作権、配信者・コメント投稿者のプライバシーを守ってください。
- 録画、文字起こし、公開は必要な権利と許可がある範囲で行ってください。
- AIによる文字起こし、要約、人物タグ、感情分析には誤りが含まれます。
- 公開前に生成HTMLと個人情報を確認してください。
